# Template Library

> **Index of document templates for consistent documentation.**

## Available Templates

| Template | Use Case | Location |
|:---------|:---------|:---------|
| [README Template](TEMPLATES/TEMPLATE_README.md) | Section navigation and overview | New directories |
| [Research Finding Template](TEMPLATES/TEMPLATE_RESEARCH_FINDING.md) | Documenting discoveries | Research outputs |
| [Case Study Template](TEMPLATES/TEMPLATE_CASE_STUDY.md) | Detailed success stories | Investor/marketing materials |
| [Domain Explainer Template](TEMPLATES/TEMPLATE_DOMAIN_EXPLAINER.md) | Audience-specific explanations | Tier 1 scientific domains |

---

## How to Use Templates

1. **Copy** the template file to your target location
2. **Rename** following naming conventions in [CONTENT_STANDARDS.md](CONTENT_STANDARDS.md)
3. **Fill in** all sections marked with `[brackets]`
4. **Remove** the template instructions (lines starting with `>`)
5. **Review** against [REVIEW_CHECKLIST.md](REVIEW_CHECKLIST.md)

## Template Descriptions

### README Template
**Purpose**: Create consistent navigation documents for directories.

**Use when**: Creating a new section or directory that needs an overview.

**Sections**:
- Purpose statement
- What's in this section
- Key documents
- Related sections

---

### Research Finding Template
**Purpose**: Document scientific discoveries in a structured format.

**Use when**: Recording a new research finding, insight, or validation result.

**Sections**:
- Executive summary
- Background & context
- Methodology
- Findings
- Implications
- Validation status
- Next steps

---

### Case Study Template
**Purpose**: Tell compelling stories of successful applications.

**Use when**: Creating materials for investors, partners, or marketing.

**Sections**:
- Challenge
- Approach
- Results
- Key metrics
- Lessons learned
- Applicability

---

### Domain Explainer Template
**Purpose**: Explain the project to specific scientific audiences.

**Use when**: Creating Tier 1 documents for clinicians, virologists, immunologists, etc.

**Sections**:
- Why this matters to [domain]
- Key predictions
- Validation protocols
- How to test our claims
- Related resources

---

## Creating New Templates

When creating a new template:

1. Identify recurring document pattern
2. Create template with:
   - Clear section headers
   - Placeholder text in `[brackets]`
   - Instructions in `> blockquotes`
3. Add to this index
4. Document in CONTENT_STANDARDS.md if needed

---

## Template Maintenance

Templates should be:
- Reviewed quarterly
- Updated when standards change
- Versioned if major changes occur
