## Structure & Format

- [ ] Single H1 title at top
- [ ] Purpose statement (blockquote under title)
- [ ] Logical heading hierarchy (H1 > H2 > H3)
- [ ] "Last updated" date at bottom
- [ ] Table of contents (if document >500 lines)

---

## Content Quality

### Clarity
- [ ] Purpose is clear in first paragraph
- [ ] Jargon is explained or avoided
- [ ] Active voice used predominantly
- [ ] Sentences are concise

### Accuracy
- [ ] All claims are accurate
- [ ] Numbers have sources or validation
- [ ] Dates and versions are current
- [ ] Technical details are correct

### Completeness
- [ ] All sections are filled (no placeholders left)
- [ ] Related documents are linked
- [ ] Next steps or call-to-action included (if appropriate)

---

## Claim Language (Critical)

### Validation Levels Used Correctly
- [ ] Level 1 ("is", "proves") - Only for mathematically proven claims
- [ ] Level 2 ("correlates with", "validated by") - Only for externally corroborated claims
- [ ] Level 3 ("suggests", "indicates") - For internally consistent claims
- [ ] Level 4 ("may", "could", "hypothesized") - For unvalidated hypotheses

### Tier-Specific Requirements

**For Tier 1 (Public) and Tier 2 (Investor) documents:**
- [ ] No forbidden terms appear:
  - [ ] "p-adic" → use "proprietary encoding"
  - [ ] "ternary" → use "specialized representation"
  - [ ] "hyperbolic" → use "geometric" or "non-Euclidean"
  - [ ] "non-Archimedean" → use "proprietary metric"
  - [ ] "3-adic" → use "specialized distance metric"
  - [ ] "Poincare ball" → use "geometric embedding space"

- [ ] All claims are Level 2 or higher (no unvalidated hypotheses)

---

## Links & References

- [ ] All internal links work (tested)
- [ ] All external links work (tested)
- [ ] Links use relative paths (not absolute)
- [ ] References are properly cited
- [ ] Image paths are correct

---

## Images & Diagrams

- [ ] All images have alt text
- [ ] Diagrams are readable at normal size
- [ ] Mermaid diagrams render correctly
- [ ] Images are in appropriate folder (assets/ or 06_DIAGRAMS/)

---

## Code Examples (if applicable)

- [ ] Code blocks have language specified
- [ ] Code is syntactically correct
- [ ] Code examples are tested
- [ ] Commands include expected output or behavior

---

## Formatting

- [ ] Tables are properly formatted
- [ ] Lists are consistent (all bullets or all numbers)
- [ ] Bold/italic used consistently
- [ ] No trailing whitespace
- [ ] Proper line breaks between sections

---

## Tier-Specific Checks

### Tier 1 (Public)
- [ ] Accessible to non-specialists
- [ ] Includes validation protocols
- [ ] Contact information for follow-up
- [ ] No confidential information

### Tier 2 (Investor)
- [ ] Business value is clear
- [ ] Market context provided
- [ ] Claims map to evidence
- [ ] Call-to-action included

### Tier 3 (Technical)
- [ ] Sufficient technical depth
- [ ] Implementation details accurate
- [ ] CONFIDENTIAL marking if required
- [ ] Access controls noted

---

## Final Checks

- [ ] Spell-checked
- [ ] Grammar-checked
- [ ] Read aloud for flow
- [ ] Reviewed by second person (for important documents)
